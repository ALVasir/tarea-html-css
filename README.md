# Proyecto HTML5 y CSS3 — 25/26

## Descripción

El proyecto consiste en realizar una web sobre alguna temática de libre elección.

1. El primer paso es elegir el contenido que se quiere mostrar.
2. El segundo es elegir el diseño, utilizando plantillas ya diseñadas. Se debe estudiar el código tanto HTML como CSS. **Se valorará positivamente el uso de hojas de estilos adaptables** para que se visualicen de manera óptima en dispositivos móviles.
3. Modificar el HTML y el CSS de la plantilla para adaptarla a las necesidades del proyecto.
4. Una vez concluida la página web, darse de alta en un servicio de hosting gratuito y subir la página.

---

## Requisitos del sitio web

| # | Requisito |
|---|-----------|
| 1 | La página debe tener **menú desplegable** y al menos **4 páginas web** enlazadas entre sí. |
| 2 | La página debe estar hecha en **HTML5** y todas las páginas deberán ser válidas. |
| 3 | La estructura del HTML debe incluir al menos **tres** de los siguientes elementos: `<header>`, `<footer>`, `<aside>`, `<section>`, `<article>`. |
| 4 | Se deben utilizar **etiquetas de texto**: títulos, párrafos, marcado de texto… |
| 5 | Deben existir **enlaces a páginas externas** al sitio web. |
| 6 | Debe existir alguna **lista ordenada y no ordenada**. |
| 7 | Se deben incluir **imágenes**. |
| 8 | Debe existir al menos **una tabla**. |
| 9 | Debe existir un **formulario** que envíe los datos a `http://endor.josedomingo.org/josedom/resultado.php`. |
| 10 | Todas las páginas deben tener un **favicon**. |

---

## Entrega

El documento PDF de entrega debe incluir:

- **Plantilla utilizada:** página donde se encontró y su nombre.
- **Cambios realizados:** descripción de las principales modificaciones sobre la plantilla.
- **Capturas de pantalla** de las cuatro páginas del sitio web servidas desde el servidor cloud.
- **URL** de la página alojada en el hosting.
- **Explicación del proceso de subida** al hosting gratuito:
  - ¿Qué hosting se ha escogido?
  - ¿Qué cliente FTP se ha utilizado?
  - ¿A qué directorio del servidor remoto se ha subido la página?
- **Capturas de validación** de cada página con el [validador W3C](https://validator.w3.org/) (debe aparecer la URL de la página en cada captura).
